1. INTRODUCTION TO DATAFLOWS GEN2
================================================================================

1.1 What Are Dataflows Gen2?

Dataflows Gen2 are a cloud-based ETL (Extract, Transform, Load) tool in Microsoft Fabric used to ingest and transform data from multiple sources, and then land the cleansed data to a destination. They are a crucial component of the end-to-end analytics data ingestion process in Microsoft Fabric.

Dataflows Gen2 use Power Query Online, which provides a visual, low-to-no-code interface to perform ETL tasks. This makes them accessible to a wide range of users — from data engineers to Power BI developers.

1.2 Why Are Dataflows Gen2 Important?

Consider a scenario where you work for a retail company with stores across the globe. You need to prepare and transform data from various sources into a format suitable for data analysis and reporting. The business requests a semantic model that consolidates disparate data sources from different stores.

Without a dataflow, you would have to manually extract and transform data from every source — a process that is time-consuming and prone to errors. Dataflows Gen2 solve this by:

  - Preparing data to ensure consistency across sources
  - Staging data in a preferred destination (like a lakehouse)
  - Enabling reuse of transformation logic
  - Making it easy to update data on schedule or on demand

1.3 Where Do Dataflows Gen2 Fit in Fabric?

Dataflows Gen2 can be:
  - Incorporated into data pipelines for more complex activity orchestration
  - Used as a data source in Power BI
  - Loaded into a new table in a lakehouse, warehouse, or SQL database

They are a unified solution for data engineering, integration, and analytics within the Fabric platform.


2. UNDERSTANDING DATAFLOWS GEN2 IN MICROSOFT FABRIC
================================================================================

2.1 What is a Dataflow?

A dataflow is a type of cloud-based ETL (Extract, Transform, Load) tool for building and executing scalable data transformation processes. Dataflows Gen2 specifically allow you to:

  - Extract data from various sources
  - Transform it using a wide range of transformation operations
  - Load it into a destination

The visual interface for performing these tasks is Power Query Online.

Fundamentally, a dataflow includes ALL of the transformations needed to reduce data prep time. Once built, a dataflow can be:
  - Loaded into a new table
  - Included in a data pipeline
  - Used as a data source by data analysts

2.2 How to Use Dataflows Gen2

There are several patterns for using Dataflows Gen2:

  PATTERN 1: Dataflow Gen2 Only (ETL)
    - Create a Dataflow Gen2 first to extract and transform data
    - Load the data directly into a lakehouse or other destination
    - The business can then consume the curated semantic model

  PATTERN 2: Pipeline with Coding (without Dataflow)
    - Use a data pipeline to copy data
    - Use your preferred coding language to extract, transform, and load the data
    - This is more manual and requires coding skills

  PATTERN 3: Pipeline + Dataflow Gen2 (ETL with Orchestration)
    - Create a Dataflow Gen2 for extraction and transformation
    - Add the dataflow as an activity in a data pipeline for additional orchestration
    - Adding a data destination to your dataflow is OPTIONAL
    - The dataflow preserves all transformation steps regardless

  PATTERN 4: Pipeline + Dataflow Gen2 for ELT (Extract, Load, Transform)
    - Use a Pipeline to extract and load raw data into a destination (like a lakehouse)
    - Then create a Dataflow Gen2 to connect to the lakehouse data for cleansing and transformation
    - Offer the Dataflow as a curated semantic model for data analysts to build reports

  Note on ETL vs ELT:
    - ETL: Extract → Transform → Load (transform before loading)
    - ELT: Extract → Load → Transform (load first, then transform in the destination)

2.3 Horizontal Partitioning of Dataflows

Dataflows can be horizontally partitioned. This means you can create a global dataflow that handles common transformations, and then data analysts can create specialized dataflows (semantic models) for specific needs on top of it.

This promotes reuse: once a global dataflow is created, multiple analysts can build on it without duplicating work.

2.4 Reusability and Discoverability

Dataflows allow you to promote reusable ETL logic that prevents the need to create additional connections to your data source. Key points:

  - Dataflows offer a wide variety of transformations
  - They can be run manually, on a refresh schedule, or as part of a data pipeline orchestration
  - You can make your dataflow DISCOVERABLE so data analysts can connect to it through Power BI Desktop
  - This reduces the data preparation work needed for report development

2.5 Benefits of Dataflows Gen2

  1. EXTEND DATA WITH CONSISTENT DATA
     You can add standard data, such as a standard date dimension table, to extend your dataset.

  2. SELF-SERVICE ACCESS
     Allow self-service users access to a subset of data warehouse data separately.

  3. OPTIMIZE PERFORMANCE
     Dataflows enable extracting data once for reuse, which reduces data refresh time for slower sources.

  4. SIMPLIFY DATA SOURCE COMPLEXITY
     Only expose dataflows to larger analyst groups, hiding the complexity of the underlying data sources.

  5. ENSURE CONSISTENCY AND QUALITY
     Enable users to clean and transform data before loading it to a destination, ensuring data quality.

  6. SIMPLIFY DATA INTEGRATION
     Provide a low-code interface that ingests data from various sources, making integration simpler.

2.6 Limitations of Dataflows Gen2

  1. NOT A REPLACEMENT FOR A DATA WAREHOUSE
     Dataflows are not designed to replace data warehouses. They are a data preparation tool.

  2. ROW-LEVEL SECURITY NOT SUPPORTED
     Dataflows Gen2 do not support row-level security.

  3. FABRIC CAPACITY WORKSPACE REQUIRED
     You need a Fabric capacity workspace to use Dataflows Gen2.


3. EXPLORING THE DATAFLOWS GEN2 INTERFACE
================================================================================

You can create a Dataflow Gen2 in the Data Factory workload, a Power BI workspace, or directly in the lakehouse. The interface uses Power Query Online to visualize transformations.

3.1 Power Query Ribbon (Component 1)

The Power Query ribbon provides access to data source connectors and transformation operations.

  Supported Data Source Connectors Include:
    - Cloud and on-premises relational databases
    - Excel or flat files
    - SharePoint
    - SalesForce
    - Spark
    - Fabric lakehouses

  Available Data Transformations Include:
    - Filter and Sort rows
    - Pivot and Unpivot
    - Merge and Append queries
    - Split and Conditional split
    - Replace values and Remove duplicates
    - Add, Rename, Reorder, or Delete columns
    - Rank and Percentage calculator
    - Choose Top N and Bottom N

  Additional ribbon features:
    - Create and manage data source connections
    - Manage parameters
    - Configure the default data destination

3.2 Queries Pane (Component 2)

The Queries pane shows the different data sources, which are called "queries." When loaded to your data store, these queries become tables.

  Key capabilities:
    - DUPLICATE a query: Creates multiple copies of the same data (useful for star schema design, splitting data into separate, smaller tables)
    - REFERENCE a query: Creates a reference to an existing query
    - DISABLE LOAD: You can disable the load of a query if you only need a one-time import

3.3 Diagram View (Component 3)

The Diagram View provides a visual representation of how data sources are connected and what transformations have been applied.

  Example flow:
    - Dataflow connects to a data source
    - Duplicates the query
    - Removes columns from the source query
    - Unpivots the duplicate query

  Each query is shown as a shape with all applied transformations, connected by lines for relationships (like duplicate queries). You can turn this view on or off.

3.4 Data Preview Pane (Component 4)

The Data Preview pane shows a SUBSET of data (not all data) to help you see:
  - Which transformations you should make
  - How transformations affect the data

  Interactive features:
    - Drag and drop columns to change order
    - Right-click on columns to filter or make changes
    - Shows all transformations for the selected query

3.5 Query Settings Pane (Component 5)

The Query Settings pane includes the Applied Steps section.

  Applied Steps:
    - Each transformation is represented as a step
    - Some steps are automatically applied when you connect a data source
    - Depending on complexity, you might have several applied steps per query
    - Most steps have a gear icon to modify them; otherwise, you must delete and repeat the transformation

  Contextual Menu (right-click on steps):
    - Rename steps
    - Reorder steps
    - Delete steps
    - View the data source query (for sources that support query folding)

  Advanced Editor:
    While the visual interface is helpful, you can also view and edit the underlying M code through the Advanced Editor.

3.6 Data Destinations

In the Query Settings pane, you can configure a Data Destination to land your data. Supported destinations within Fabric include:

  - Lakehouse
  - Warehouse
  - SQL database

  External destinations also supported:
  - Azure SQL Database
  - Azure Data Explorer
  - Azure Synapse Analytics

  Important: Adding a data destination is OPTIONAL. The dataflow preserves all transformation steps regardless of whether a destination is set.


4. INTEGRATING DATAFLOWS GEN2 AND PIPELINES
================================================================================

4.1 Why Combine Dataflows and Pipelines?

Dataflows Gen2 provide excellent data transformation capabilities. However, when you need to perform ADDITIONAL operations on the transformed data, combining dataflows with pipelines becomes powerful.

The combination is useful when you need orchestration — running activities in a specific order with dependencies.

4.2 Data Pipeline Activities

Data pipelines are a common concept in data engineering and offer a wide variety of activities to orchestrate. Common pipeline activities include:

  - Copy data
  - Incorporate Dataflow (add a Dataflow Gen2 as an activity)
  - Add Notebook
  - Get metadata
  - Execute a script or stored procedure

4.3 How to Use Them Together

The typical workflow:
  1. Use a dataflow for data ingestion, transformation, and landing into a Fabric data store
  2. Incorporate the dataflow into a pipeline to orchestrate additional activities
  3. For example: After the dataflow completes, execute scripts or stored procedures

  Pipelines provide a visual way to complete activities in a specific order.

4.4 Scheduling and Automation

Pipelines can be:
  - Scheduled to run at specific times
  - Activated by a trigger

  This means your dataflow can be refreshed automatically when needed, instead of having to manually run it.

  Why this matters: When dealing with enterprise data or frequently changing data, automation allows you to focus on other responsibilities rather than manually running data refreshes.
